# SQUIZ 외부 서비스 정보

> 이 문서는 프로젝트에서 사용하는 외부 서비스(OAuth, API 등)의 가입 및 설정 방법을 정리합니다.

---

## 1. 소셜 로그인 (OAuth 2.0)

### 1.1 카카오 (Kakao)

**용도**: 소셜 로그인

**가입 및 설정**:
1. [Kakao Developers](https://developers.kakao.com/) 접속
2. 애플리케이션 추가하기
3. 앱 키 → REST API 키 복사
4. 제품 설정 → 카카오 로그인 활성화
5. Redirect URI 등록:
   - 로컬: `http://localhost:3000/login/oauth2/callback/kakao`
   - 운영: `https://modustudy.site/login/oauth2/callback/kakao`
6. 동의항목 설정: 닉네임, 프로필 사진, 이메일 (선택)
7. 보안 → Client Secret 생성

**환경 변수**:
```env
KAKAO_REST_API_KEY=<REST API 키>
KAKAO_CLIENT_SECRET_KEY=<Client Secret>
```

**참고 문서**: https://developers.kakao.com/docs/latest/ko/kakaologin/common

---

### 1.2 네이버 (Naver)

**용도**: 소셜 로그인

**가입 및 설정**:
1. [Naver Developers](https://developers.naver.com/) 접속
2. Application → 애플리케이션 등록
3. 사용 API: 네이버 로그인 선택
4. 제공 정보: 이름, 이메일, 프로필 사진
5. 서비스 URL 및 Callback URL 등록:
   - 로컬: `http://localhost:3000/login/oauth2/callback/naver`
   - 운영: `https://modustudy.site/login/oauth2/callback/naver`
6. Client ID, Client Secret 확인

**환경 변수**:
```env
NAVER_CLIENT_ID=<Client ID>
NAVER_CLIENT_SECRET=<Client Secret>
```

**참고 문서**: https://developers.naver.com/docs/login/overview/overview.md

---

### 1.3 구글 (Google)

**용도**: 소셜 로그인 + Google Calendar 연동

**가입 및 설정**:
1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 새 프로젝트 생성
3. API 및 서비스 → 사용자 인증 정보
4. OAuth 2.0 클라이언트 ID 생성 (웹 애플리케이션)
5. 승인된 리디렉션 URI:
   - 로컬: `http://localhost:3000/login/oauth2/callback/google`
   - 운영: `https://modustudy.site/login/oauth2/callback/google`
6. OAuth 동의 화면 구성
   - 앱 이름, 로고, 개인정보처리방침 URL 등록
   - 범위 추가: `email`, `profile`, `openid`

**환경 변수**:
```env
GOOGLE_CLIENT_ID=<Client ID>
GOOGLE_CLIENT_SECRET=<Client Secret>
```

**참고 문서**: https://developers.google.com/identity/protocols/oauth2

## 2. Google Calendar API

**용도**: 사용자 캘린더와 스터디 일정 연동

**추가 설정**:
1. Google Cloud Console → API 및 서비스 → 라이브러리
2. "Google Calendar API" 검색 및 사용 설정
3. OAuth 동의 화면에서 범위 추가:
   - `https://www.googleapis.com/auth/calendar`
   - `https://www.googleapis.com/auth/calendar.events`

**사용 라이브러리** (Backend):
```gradle
implementation 'com.google.apis:google-api-services-calendar:v3-rev20231123-2.0.0'
implementation 'com.google.auth:google-auth-library-oauth2-http:1.20.0'
```

**참고 문서**: https://developers.google.com/calendar/api/quickstart/java

## 3. Firebase (FCM 푸시 알림)

**용도**: 모바일 앱 푸시 알림

**가입 및 설정**:
1. [Firebase Console](https://console.firebase.google.com/) 접속
2. 프로젝트 추가
3. 앱 추가 → Android 선택
4. 패키지 이름: `com.ssafy.squiz`
5. `google-services.json` 다운로드 → `mobile/app/` 에 배치
6. 프로젝트 설정 → 서비스 계정 → 새 비공개 키 생성
7. 생성된 JSON 파일을 Backend에서 사용

**Backend 설정**:
```java
// Firebase Admin SDK 초기화
FileInputStream serviceAccount = new FileInputStream("firebase-service-account.json");
FirebaseOptions options = FirebaseOptions.builder()
    .setCredentials(GoogleCredentials.fromStream(serviceAccount))
    .build();
FirebaseApp.initializeApp(options);
```

**사용 라이브러리**:
```gradle
implementation 'com.google.firebase:firebase-admin:9.2.0'
```

**참고 문서**: https://firebase.google.com/docs/cloud-messaging

## 4. Claude API (Anthropic)

**용도**: AI 회의 요약, 퀴즈 생성, 커리큘럼 생성

**가입 및 설정**:
1. [Anthropic Console](https://console.anthropic.com/) 접속
2. 회원가입 및 API 키 생성
3. 사용량 확인 및 결제 정보 등록

**환경 변수** (AI Server):
```env
CLAUDE_API_KEY=sk-ant-api03-xxxxx
CLAUDE_API_URL=https://api.anthropic.com/v1/messages
```

**API 사용 예시**:
```python
import httpx

response = httpx.post(
    "https://api.anthropic.com/v1/messages",
    headers={
        "x-api-key": CLAUDE_API_KEY,
        "anthropic-version": "2023-06-01",
        "content-type": "application/json"
    },
    json={
        "model": "claude-3-sonnet-20240229",
        "max_tokens": 4096,
        "messages": [{"role": "user", "content": "..."}]
    }
)
```

**요금**: https://www.anthropic.com/pricing
- Claude 3 Sonnet: $3 / 1M input tokens, $15 / 1M output tokens

**참고 문서**: https://docs.anthropic.com/claude/reference/getting-started-with-the-api

## 5. 이메일 서비스 (Gmail SMTP)

**용도**: 비밀번호 재설정, 알림 이메일 발송

**설정 방법**:
1. Google 계정 → 보안 → 2단계 인증 활성화
2. 앱 비밀번호 생성 (메일 선택)
3. 생성된 16자리 비밀번호 사용

**환경 변수**:
```env
MAIL_USERNAME=your_email@gmail.com
MAIL_PASSWORD=xxxx xxxx xxxx xxxx  # 앱 비밀번호 (공백 포함)
```

**Spring Boot 설정** (`application.yml`):
```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: ${MAIL_USERNAME}
    password: ${MAIL_PASSWORD}
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```

## 7. 로컬 개발 시 주의사항

### 7.1 OAuth Redirect URI

로컬 개발 시에는 각 OAuth 서비스에 `localhost` 주소를 등록해야 합니다:
- `http://localhost:3000/login/oauth2/callback/kakao`
- `http://localhost:3000/login/oauth2/callback/naver`
- `http://localhost:3000/login/oauth2/callback/google`

### 7.2 HTTPS 요구사항

일부 OAuth 서비스는 운영 환경에서 HTTPS를 요구합니다. 로컬에서는 HTTP로 테스트 가능합니다.

### 7.3 API 키 보안

- `.env` 파일은 절대 Git에 커밋하지 않습니다.
- `.gitignore`에 `.env` 파일이 포함되어 있는지 확인합니다.
- 운영 환경에서는 환경 변수 또는 시크릿 매니저를 사용합니다.
