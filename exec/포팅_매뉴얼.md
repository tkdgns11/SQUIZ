# SQUIZ 포팅 매뉴얼

> 이 문서는 SQUIZ 프로젝트를 로컬 환경에서 빌드 및 실행하기 위한 가이드입니다.

---

## 1. 프로젝트 개요

### 1.1 서비스 구성

| 서비스 | 설명 | 포트 |
|--------|------|------|
| Backend | Spring Boot API 서버 | 8080 |
| Frontend | React SPA | 3000 |
| SFU Server | WebRTC 미디어 서버 (mediasoup) | 4000 |
| AI Server | AI 추론 서버 (STT, 요약) | 8000 |
| MySQL | 데이터베이스 | 3306 |
| Redis | 세션/캐시 저장소 | 6379 |

### 1.2 아키텍처

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Frontend   │────▶│   Backend   │────▶│    MySQL    │

└─────────────┘     └──────┬──────┘     └─────────────┘
                          │
                    ┌─────▼─────┐
                    │   Redis   │

┌─────────────┐     ┌─────────────┐
│ SFU Server  │────▶│  AI Server  │
│ (mediasoup) │     │  (FastAPI)  │

```

## 4. 로컬 빌드 및 실행

### 4.1 방법 1: Docker Compose (권장)

가장 간단한 방법입니다. MySQL, Redis, Monitor가 한번에 실행됩니다.

```bash
# 1. 프로젝트 루트로 이동
cd modustudy

# 2. Docker 네트워크 생성 (최초 1회)
docker network create squiz-network

# 3. .env 파일 생성 (Docker용)
cat > .env << EOF
DB_ROOT_PASSWORD=root
DB_USERNAME=ssafy
DB_PASSWORD=ssafy
MONITOR_API_KEY=squiz-monitor-secret
EOF

# 4. 인프라 서비스 실행
docker-compose up -d

# 5. 서비스 상태 확인
docker-compose ps
```

### 4.2 방법 2: 개별 실행

#### 4.2.1 MySQL 실행

```bash
# Docker로 실행
docker run -d \
  --name squiz-mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=squiz \
  -e MYSQL_USER=ssafy \
  -e MYSQL_PASSWORD=ssafy \
  -e TZ=Asia/Seoul \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci \
  --default-time-zone='+09:00'
```

#### 4.2.2 Redis 실행

```bash
docker run -d \
  --name squiz-redis \
  -p 6379:6379 \
  redis:alpine
```

#### 4.2.3 Backend 실행

```bash
cd modustudy/backend

# 의존성 설치 및 빌드
./gradlew build -x test

# 개발 서버 실행
./gradlew bootRun

# 또는 JAR 직접 실행
java -jar build/libs/backend-1.0-SNAPSHOT.jar
```

#### 4.2.4 Frontend 실행

```bash
cd modustudy/frontend

# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
```

#### 4.2.5 SFU Server 실행

```bash
cd modustudy/sfu-server

# 의존성 설치
npm install

# 서버 실행
npm start
```

#### 4.2.6 AI Server 실행 (선택)

```bash
cd modustudy/ai-inference

# 가상환경 생성
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 의존성 설치
pip install -r requirements.txt

# 서버 실행
python inference_server.py
```

## 5. 데이터베이스 설정

### 5.1 스키마 마이그레이션

Flyway가 자동으로 마이그레이션을 실행합니다. Backend 실행 시 자동 적용됩니다.

수동 실행:
```bash
cd modustudy/backend
./gradlew flywayMigrate
```

### 5.2 시드 데이터

필요한 기본 데이터 삽입:

```bash
# MySQL 접속
mysql -u ssafy -p squiz

# 시드 데이터 실행
source docs/sql/region_seed_data.sql
source docs/sql/quiz_course_generated_data(sol).sql
```

제공된 `DB_덤프.sql` 파일 사용:

```bash
mysql -u ssafy -p squiz < exec/DB_덤프.sql
```

## 6. SSL 인증서 설정 (HTTPS 로컬 개발)

WebRTC 테스트를 위해 HTTPS가 필요합니다.

### 6.1 mkcert 설치

```bash
# macOS
brew install mkcert
mkcert -install

# Windows (Chocolatey)
choco install mkcert
mkcert -install

# Ubuntu
sudo apt install libnss3-tools
wget https://github.com/FiloSottile/mkcert/releases/download/v1.4.4/mkcert-v1.4.4-linux-amd64
sudo mv mkcert-v1.4.4-linux-amd64 /usr/local/bin/mkcert
sudo chmod +x /usr/local/bin/mkcert
mkcert -install
```

### 6.2 인증서 생성

```bash
# 프로젝트 루트에서
mkcert -cert-file modustudy.local.pem -key-file modustudy.local-key.pem localhost 127.0.0.1 modustudy.local

# hosts 파일에 추가 (관리자 권한)
# Windows: C:\Windows\System32\drivers\etc\hosts
# Linux/Mac: /etc/hosts
127.0.0.1 modustudy.local
```

## 9. 트러블슈팅

### 9.1 Java 버전 오류

```
Error: Java 21 is required
```

해결:
```bash
# JAVA_HOME 설정
export JAVA_HOME=/path/to/jdk-21
export PATH=$JAVA_HOME/bin:$PATH
```

### 9.2 Node.js 버전 오류

```bash
# nvm 사용
nvm install 20
nvm use 20
```

### 9.3 mediasoup 빌드 오류

mediasoup은 C++ 컴파일이 필요합니다:

```bash
# Ubuntu
sudo apt-get install build-essential python3

# macOS
xcode-select --install

# Windows
npm install --global windows-build-tools
```

### 9.4 MySQL 한글 깨짐

```sql
ALTER DATABASE squiz CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 9.5 Redis 연결 실패

```bash
# Redis 컨테이너 확인
docker ps | grep redis

# 로그 확인
docker logs squiz-redis
```
