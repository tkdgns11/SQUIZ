## 2. 핵심 사용자 동선 (User Journey)

### Journey 1: 스터디 찾기 → 가입 (이지은)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    스터디 참여자의 여정 (이지은)                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  [1. 탐색]          [2. 상세 확인]      [3. 신청]         [4. 활동]      │
│                                                                          │
│  ┌─────────┐      ┌──────────┐       ┌──────────┐      ┌──────────┐    │
│  │ 스터디  │      │ 스터디장 │       │ 상세보기 │      │ 팀 활동  │    │
│  │ 탐색    │─────▶│ 평점/    │──────▶│ + 신청   │─────▶│ 시작     │    │
│  │ 필터링  │      │ 리뷰확인 │       │          │      │          │    │
│  └─────────┘      └──────────┘       └──────────┘      └──────────┘    │
│       │                │                   │                 │          │
│       ▼                ▼                   ▼                 ▼          │
│  지역/요일/시간대  스터디장 운영 이력    댓글로 질문      승인 알림 →    │
│  카테고리 필터     + 리뷰 확인          자기소개 작성      팀 입장        │
│  찜하기 기능                                                              │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

**상세 흐름:**

| 단계 | 화면 | 사용자 행동 | 시스템 반응 |
|-----|------|------------|------------|
| 1 | 스터디 탐색 | 필터 설정 (지역/요일/시간대/카테고리) | 조건에 맞는 스터디 목록 표시 |
| 2 | 스터디 목록 | 관심 스터디 찜하기 | 찜 목록에 저장 |
| 3 | 스터디 상세 | 스터디장 클릭 | 스터디장 프로필 + 운영한 스터디 + 리뷰 표시 |
| 4 | 스터디 상세 | 댓글로 질문 | 댓글 등록, 스터디장에게 알림 |
| 5 | 스터디 상세 | "가입 신청" 클릭 | 신청 모달 표시 |
| 6 | 신청 모달 | 자기소개 작성 후 제출 | "신청 완료" 알림 |
| 7 | 내 신청 목록 | 신청 현황 확인 | 대기중/승인/거절 상태 표시 |
| 8 | - | (대기) | 팀장이 승인 시 "승인됨" 알림, 거절 시 사유 안내 |
| 9 | 알림 | 알림 클릭 | 팀 메인 화면으로 이동 |

### Journey 2: 스터디 모집 → 운영 (김민수)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    스터디 리더의 여정 (김민수)                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  [1. 모집글 작성]  [2. 신청 관리]   [3. 모집 완료]      [4. 운영]        │
│                                                                          │
│  ┌─────────┐  ┌──────────┐    ┌──────────┐       ┌──────────┐          │
│  │ 모집글  │  │ 신청자   │    │ 정식채널 │       │ 팀 운영  │          │
│  │ 작성    │─▶│ 심사     │───▶│ 생성     │──────▶│ 시작     │          │
│  └─────────┘  └──────────┘    └──────────┘       └──────────┘          │
│       │            │               │                  │                 │
│       ▼            ▼               ▼                  ▼                 │
│  템플릿 활용    자기소개 확인   인원충족→정식채널   커리큘럼 설정        │
│  상세 정보 입력 승인/거절+사유  인원부족→임시채널   일정 관리            │
│                                연장 또는 취소                            │
└─────────────────────────────────────────────────────────────────────────┘

**상세 흐름:**

| 단계 | 화면 | 사용자 행동 | 시스템 반응 |
|-----|------|------------|------------|
| 1 | 모집글 작성 | 템플릿 선택 (알고리즘/CS/면접 등) | 템플릿 기본 양식 로드 |
| 2 | 모집글 작성 | 상세 정보 입력 (지역, 요일, 시간, 인원 등) | 모집글 등록 |
| 3 | - | (대기) | 신청자 발생 시 알림 |
| 4 | 신청 관리 | 신청자 자기소개 확인 | 승인/거절 (거절 시 사유 필수) |
| 5 | - | (모집 기간 종료) | 인원 충족 → 정식 채널 생성, 부족 → 임시 채널 |
| 6 | 모집 연장 | 인원 부족 시 연장 결정 (1회 제한) | 연장 또는 취소/진행 선택 |
| 7 | 정식 전환 | 진행 결정 시 | 임시 → 정식 채널 전환 |
| 8 | 팀 설정 | 커리큘럼, 일정 등록 | 팀원에게 알림 |

## 4. MVP 우선순위별 동선

### P0 (필수) - 핵심 기능

```
[회원가입/로그인] → [스터디 탐색] → [가입 신청] → [팀 입장]
                                         ↓
[모집글 작성] → [신청 관리] → [모집 완료] → [팀 운영]
                                         ↓
                         [음성 채널] → [미팅 시작] → [AI 요약]
                                         ↓
                         [텍스트 채팅] → [자료 공유] → [커리큘럼]
```
