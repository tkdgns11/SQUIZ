# ModuStudy 스터디 상태 관리

## 상태 정의

| 상태 | 코드 | 설명 |
|------|------|------|
| 임시저장 | `DRAFT` | 모집글 작성 중 (미공개) |
| 모집예정 | `SCHEDULED` | 모집 시작일 전 |
| 모집중 | `RECRUITING` | 모집 진행 중 |
| 모집완료 | `RECRUIT_CLOSED` | 모집 마감 + 인원 충족 → 시작 대기 |
| 확정대기 | `PENDING` | 모집 마감 + 인원 부족 → 진행 여부 결정 대기 |
| 진행중 | `IN_PROGRESS` | 스터디 진행 중 |
| 완료 | `COMPLETED` | 스터디 정상 종료 |
| 취소 | `CANCELLED` | 스터디 취소 |

## 상태 전환 조건

### DRAFT → SCHEDULED
- 모집글 작성 완료 후 등록
- 모집 시작일이 미래인 경우

### DRAFT → RECRUITING
- 모집글 작성 완료 후 등록
- 모집 시작일이 오늘이거나 과거인 경우

### SCHEDULED → RECRUITING
- 모집 시작일 도래 (자동)

### RECRUITING → RECRUIT_CLOSED
- 모집 마감일 도래 + 인원 충족 (자동)
- 조기 모집 종료 (스터디장 수동)
- **정식 채널 자동 생성**

### RECRUITING → PENDING
- 모집 마감일 도래 + 인원 부족 (자동)
- **임시 채널 자동 생성** (텍스트 전용)
- 대기중 신청자 자동 거절 처리

### PENDING → RECRUITING
- 모집 연장 (1회 제한)
- 스터디 일정 재등록 필요

### PENDING → IN_PROGRESS
- 스터디장이 현재 인원으로 진행 결정
- 임시 채널 → 정식 채널 전환
- **기존 대화 내역 삭제**

### PENDING → CANCELLED
- 스터디장이 취소 결정

### RECRUIT_CLOSED → IN_PROGRESS
- 스터디 시작일 도래 (자동)
- 온라인 스터디: 미팅 자동 시작

### IN_PROGRESS → COMPLETED
- 스터디 종료일 도래 (자동)
- 스터디장 수동 완료 처리

### IN_PROGRESS → CANCELLED
- 스터디장이 중도 취소

## 채널 정책

| 상태 | 채널 | 기능 |
|------|------|------|
| RECRUITING | 없음 | 채널 미생성 |
| RECRUIT_CLOSED | 정식 채널 | 텍스트/음성/미팅/자료실/출석 |
| PENDING | 임시 채널 | 텍스트만 (진행 여부 논의용) |
| IN_PROGRESS | 정식 채널 | 모든 기능 |

### 임시 채널 → 정식 채널 전환
- 스터디장이 "진행" 결정 시 전환
- **대화 내역 삭제** (새 멤버가 "진행할까?" 논의 내용 보지 않도록)
- 전환 후 모든 기능 활성화

## 알림 정책

| 이벤트 | 대상 | 알림 내용 |
|--------|------|----------|
| 가입 신청 | 스터디장 | "OOO님이 가입 신청했습니다" |
| 신청 수락 | 신청자 | "스터디에 수락되었습니다" |
| 신청 거절 | 신청자 | "거절되었습니다 + 사유" |
| 모집 마감 자동 거절 | 대기중 신청자 | "모집 마감으로 거절되었습니다" |
| 모집글 삭제 후 재생성 | 확정자 + 대기중 신청자 | "새 모집글이 등록되었습니다" (거절자 제외) |
| 추방 예고 | 추방 대상자 | "3일 후 추방 예정입니다" |
| 스터디 시작 | 모든 멤버 | "스터디가 시작되었습니다" |

---

## 모집 연장 규칙

1. **1회만 가능** - 추가 연장 불가
2. 연장 시 **스터디 일정 재등록** 필수
3. 연장 후에도 인원 부족 시:
   - 현재 인원으로 진행 또는
   - 스터디 취소 결정

| 자동 출석 | 미팅 시작 후 10분 내 입장 | 자동 출석 체크 |
