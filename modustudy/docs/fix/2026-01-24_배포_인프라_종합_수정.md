## 1. Profile 테이블 컬럼 누락 문제

### 증상
```
GET /api/v1/friends/search?keyword=박지원 500 (Internal Server Error)
Caused by: java.sql.SQLSyntaxErrorException: Unknown column 'p.profile_image_url' in 'field list'
```

### 원인
- `V1__init.sql`에는 `profile` 테이블에 여러 컬럼이 정의되어 있음
- 하지만 `Profile.java` 엔티티가 비어있어서 `ddl-auto=update` 실행 시 컬럼이 생성되지 않음
- MyBatis 쿼리에서 `profile_image_url` 등의 컬럼을 참조하여 에러 발생

### 해결
`V4__fix_profile_table_columns.sql` 마이그레이션 파일 생성:
```sql
-- user_id, profile_image_url, profile_image_source, bio, social_links, tech, favorite 컬럼 추가
-- 이미 존재하는 경우 스킵하도록 조건문 사용
```

### 관련 파일
- `backend/src/main/resources/db/migration/V4__fix_profile_table_columns.sql` (신규)

## 3. Socket.io WebSocket 연결 실패 문제

### 증상
```
WebSocket connection to 'wss://i14d106.p.ssafy.io/socket.io/?EIO=4&transport=websocket' failed
```

### 원인 분석
1. **Nginx 설정 누락**: `/socket.io/` 경로에 대한 프록시 설정이 없었음
2. **SFU 서버 HTTPS 모드**: SFU 서버가 HTTPS로 실행되는데, Nginx는 HTTP로 연결 시도

### 연결 흐름
```
Frontend (sfuClient.ts)
    ↓ io(baseUrl) - socket.io는 기본적으로 /socket.io/ 경로 사용
    ↓
Nginx (/socket.io/ → http://sfu:4000)
    ↓
SFU Server (port 4000)
```

### 해결

#### 3-1. Nginx에 `/socket.io/` location 추가
```nginx
# nginx/conf.d/default.conf
location /socket.io/ {
    proxy_pass http://sfu;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 86400;
    proxy_send_timeout 86400;
}
```

#### 3-2. SFU 서버 HTTPS 비활성화
```yaml
# docker-compose.blue.yml / docker-compose.green.yml
sfu-blue:  # 또는 sfu-green
  environment:
    - NODE_ENV=production
    - SFU_USE_HTTPS=false  # nginx가 SSL 처리
    - SFU_ANNOUNCED_IP=i14d106.p.ssafy.io
```

### 관련 파일
- `nginx/conf.d/default.conf`
- `docker-compose.blue.yml`
- `docker-compose.green.yml`

## 4. SFU Dockerfile 포트 불일치

### 증상
- Dockerfile의 EXPOSE 포트와 실제 사용 포트가 다름

### 원인
```dockerfile
# 기존 (잘못됨)
EXPOSE 3000
EXPOSE 40000-40100/udp
```

실제 사용:
- Socket.io: 4000 포트
- WebRTC UDP: 20000-22000 포트

### 해결
```dockerfile
# 수정 후
EXPOSE 4000
EXPOSE 20000-22000/udp
```

### 관련 파일
- `sfu-server/Dockerfile`

## 6. 아키텍처 정리

### WebSocket 연결 구조
```
┌─────────────┐     HTTPS/WSS     ┌─────────────┐     HTTP     ┌─────────────┐
│   Browser   │ ───────────────→  │    Nginx    │ ──────────→  │ SFU Server  │
│  (Frontend) │                   │   (Proxy)   │              │ (Socket.io) │
└─────────────┘                   └─────────────┘              └─────────────┘
      │                                  │
      │ /socket.io/*                     │ proxy_pass http://sfu:4000
      │ (WebSocket Upgrade)              │ (WebSocket Upgrade headers)
      └──────────────────────────────────┘
```

### Nginx → 서비스 라우팅
| 경로 | 대상 서비스 | 용도 |
|------|-------------|------|
| `/` | frontend (nginx-blue/green) | React SPA |
| `/api/` | backend (8080) | REST API |
| `/ws/` | backend (8080) | STOMP WebSocket (채팅, 알림) |
| `/socket.io/` | sfu (4000) | Socket.io WebSocket (WebRTC 시그널링) |
| `/sfu/` | sfu (4000) | SFU 헬스체크 등 |
| `/api/ai/` | cs-quiz-ai (5000) | AI 서비스 |
