# 얼굴 인식(자리있음/자리없음) 감지 문제 분석 및 해결

## 배경
미팅 화면에서는 얼굴 인식을 통해 “자리있음/자리없음” 상태를 계산하고, 참가자에게 해당 상태를 표시합니다.  
얼굴 인식은 브라우저에서 카메라 영상 프레임을 받아 TensorFlow 기반의 모델(coco-ssd)로 사람(person) 객체를 탐지하는 방식입니다.

## 기존 동작 방식
- **감지 대상 영상**
  - `MeetingRoomPage.tsx`에서 숨겨진 `<video>` 요소(`aiVideoRef`)에 카메라 스트림을 연결
  - `aiDetection.startDetection()`이 일정 주기(기본 2초)로 `detectPerson()`을 호출
- **감지 로직**
  - `coco-ssd` 모델로 프레임을 분석하고 “사람(person)”을 찾으면 자리있음으로 판단
  - 결과가 바뀌면 WebSocket을 통해 참가자 상태(자리있음/자리없음)를 갱신

## 문제 증상
1) 상대가 발표자가 되거나 발표 권한이 바뀌는 순간  
   - 내 쪽에서 얼굴 인식이 멈춤
   - “자리없음”으로 고정되고 더 이상 갱신되지 않음

2) 새로고침 시에만 정상 복구  
   - 즉, 감지 루프가 **중간에 끊겼고** 자동 복구가 없었던 상태

## 원인 분석
1) **감지 루프가 중단되거나 스트림이 끊김**
   - 발표자 전환/공유 모드 전환에서 카메라 스트림이 교체되거나 종료되는 경우가 있음
   - 기존 감지 루프가 끊겨도 자동으로 복구되지 않음

2) **document.hidden 조건**
   - 기존 로직은 `document.hidden`일 때 감지를 스킵
   - 탭/권한 팝업/발표자 전환 과정에서 hidden 상태가 되면 감지가 멈추고,  
     다시 살아나지 않는 상황이 발생

## 해결 방식(최종)
아래 3가지 방식을 동시에 적용하여 안정성을 확보했습니다.

### 1) 2초 주기 감지 유지
- 감지 루프는 2초 간격으로 계속 동작
- “일시적으로 끊겼다가 복구되는 상황”에서도 자동으로 다시 감지

### 2) hidden 체크 제거
- `document.hidden` 조건 제거
- 백그라운드/전환 상황에서도 감지 시도가 계속됨

### 3) 복구 로직 + watchdog 추가
- 감지 스트림이 `ended` / `inactive` 되면:
  - 감지 루프를 정리
  - 스트림을 재요청(getUserMedia)
  - 감지를 다시 시작
- 추가로 **watchdog 타이머**(4초 간격)를 두어  
  감지 루프가 끊겼는지 감시하고 자동 재시작

## 적용된 주요 변경점
### 1) 감지 스트림 재시작 로직
- 감지용 스트림이 종료되면 자동 재요청
- 감지 루프가 끊겨도 즉시 복구 가능

### 2) 감지 루프 로그 추가
- 감지 시작/중단/비디오 준비 상태/오류 로그를 추가하여  
  어디에서 끊기는지 추적 가능

### 3) watchdog 추가
- 감지 루프가 사라진 경우 자동 복구

## 기대 효과
1) 발표자 전환/공유 모드 전환 후에도 감지 루프 유지
2) “자리없음 고정” 현상 제거
3) 새로고침 없이 자동 복구
