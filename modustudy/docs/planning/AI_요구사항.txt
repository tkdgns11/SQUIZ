1. 미팅 AI (STT/요약/분석)
================================================================================
기능: 미팅 녹음 후 텍스트 변환 및 자동 요약 (실시간 자막 X)

[핵심 기능]
- U-FR-073: 미팅 STT - 미팅 종료 후 녹음 파일을 텍스트로 변환
- U-FR-074: 화자 분리 - 발언자별로 텍스트 분리
- U-FR-076: 미팅 요약 - 미팅 종료 후 전체 대화 내용 요약
- U-FR-077: 발언자별 요약 - 각 참가자가 말한 내용 개별 요약
- U-FR-078: 액션 아이템 추출 - 해야 할 일 자동 추출
- U-FR-217: 회의록 자동 생성 - 정형화된 회의록 형식으로 문서 생성

[사용 기술]
- Speech-to-Text (STT): Whisper (녹음 파일 후처리)
- LLM 요약: GPT-4 / Claude
- 화자 분리: Speaker Diarization

================================================================================
2. 데일리/회고 AI
================================================================================
기능: 데일리 스탠드업 및 회고 자동화

[핵심 기능]
- U-FR-091: 자동 분류 - 발언을 "어제 한 일/오늘 할 일/블로커"로 분류
- U-FR-092: 데일리 리포트 생성 - 팀 데일리 리포트 자동 생성
- U-FR-093: 블로커 하이라이트 - 블로커(장애물) 강조 표시
- U-FR-105: 회고 초안 생성 - 데일리 기록, 채팅 분석하여 회고 초안 생성
- U-FR-106: AI 패턴 분석 - 과거 회고 분석하여 팀 성장 패턴 리포트

[사용 기술]
- LLM 분류/요약: GPT-4 / Claude
- 텍스트 분석: NLP 기반 키워드 추출

================================================================================
3. 추천 AI
================================================================================
기능: 맞춤형 추천 서비스

[핵심 기능]
- U-FR-018: 스터디 추천 - 관심사, 기술스택, 일정 기반 맞춤 스터디 추천
- U-FR-212: 일정 추천 - 팀원 캘린더 분석하여 최적 미팅 시간 TOP 3 추천
- U-FR-213: 일정 추천 이유 - 추천 시간에 대한 설명 제공
- U-FR-214: 학습 추천 - 팀 활동 패턴 분석하여 학습 자료/다음 스텝 추천
- M-FR-131 (모바일): 스터디 추천 - 내 일정에 맞는 스터디 추천

[사용 기술]
- 추천 알고리즘: Collaborative Filtering / Content-based
- 캘린더 분석: 일정 최적화 알고리즘
- LLM 추천 이유 생성

2. 파인튜닝 대상
================================================================================

[2-1] Whisper 한국어 도메인 파인튜닝
--------------------------------------------------------------------------------
- 모델: openai/whisper-large-v3
- 목적: 개발 용어 인식률 향상 (PR, 머지, 커밋, API 등)
- 데이터:
  * AI Hub 한국어 음성 데이터
  * 자체 수집 개발 미팅 녹음
- 학습: Hugging Face Transformers + PEFT
- 적용: U-FR-073 미팅 STT

[2-2] LLM 요약 파인튜닝 (LoRA)
--------------------------------------------------------------------------------
- 모델: Qwen/Qwen2.5-7B-Instruct
- 목적: 미팅/데일리 요약 포맷 특화
- 데이터:
  * 미팅 전문 + 좋은 요약 쌍 (100~500개)
  * 데일리 발언 + 분류된 결과 쌍
- 학습: LoRA (rank=16, alpha=32)
- 적용: U-FR-076 미팅 요약, U-FR-092 데일리 리포트

[2-3] 텍스트 분류기 파인튜닝
--------------------------------------------------------------------------------
- 모델: monologg/koelectra-base-v3-discriminator
- 목적: 어제/오늘/블로커 3-class 분류
- 데이터: 데일리 발언 + 라벨 (500~1000개)
- 학습: Classification Head 추가 + Fine-tuning
- 적용: U-FR-091 자동 분류

[2-4] 감정 분석 파인튜닝
--------------------------------------------------------------------------------
- 모델: monologg/koelectra-base-v3-discriminator
- 목적: 팀 분위기 분석 (긍정/중립/부정)
- 데이터: 채팅/미팅 텍스트 + 감정 라벨
- 적용: U-FR-218 팀 건강도

================================================================================
3. RAG 시스템 구축
================================================================================

[구성요소]
- Vector DB: Milvus (또는 ChromaDB/Qdrant)
- Embedding: BAAI/bge-m3 (다국어 지원)
- Reranker: BAAI/bge-reranker-v2-m3 (정확도 향상)
- LLM: Qwen2.5-7B (답변 생성)

[저장 데이터]
- 미팅 녹취록 + 요약
- 회고 기록 (KPT)
- 채팅 히스토리
- 학습 자료 (링크, 문서)

[RAG 파이프라인]
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Query   │ -> │ Embedding│ -> │ Vector   │ -> │ Top-K    │
│  입력    │    │ (bge-m3) │    │ Search   │    │ Retrieved│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                      │
┌──────────┐    ┌──────────┐    ┌──────────┐         │
│  Answer  │ <- │   LLM    │ <- │ Reranker │ <───────┘
│  생성    │    │ (Qwen2.5)│    │ (bge-re) │
└──────────┘    └──────────┘    └──────────┘

[적용 기능]
- U-FR-105: 회고 초안 생성 (과거 회고 참조)
- U-FR-106: 패턴 분석 (반복 문제 탐지)
- U-FR-214: 학습 추천 (자료 기반)

[서버 스펙 (최소)]
- GPU: RTX 4090 1대 또는 A100 40GB 1대
- Qwen2.5-7B: ~14GB VRAM (FP16)
- Whisper Large: ~3GB VRAM
- 분류 모델: ~1GB VRAM

| 패턴 분석 | RAG + LLM | bge-m3 + Qwen2.5-7B |

6. 개발 우선순위
================================================================================

[Phase 1 - 기본 인프라]
1. vLLM으로 Qwen2.5-7B 추론 서버 구축
2. Whisper 추론 파이프라인 구축
3. Milvus + bge-m3 RAG 기본 구축

[Phase 2 - 파인튜닝]
4. Whisper 한국어 도메인 파인튜닝
5. Qwen2.5 요약 LoRA 파인튜닝
6. KoELECTRA 분류기 파인튜닝

[Phase 3 - 고도화]
7. 화자 분리 파이프라인 추가
8. RAG Reranker 적용
9. 감정 분석 모델 추가
