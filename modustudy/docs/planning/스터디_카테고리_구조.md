# ModuStudy - 스터디 카테고리 구조

## 개요

스터디는 **주제(Topic)**와 **형식(Format)** 두 가지 축으로 분류된다.

```
스터디 = 주제(무엇을) + 형식(어떻게)

예시:
- 알고리즘 + 문제 풀이 = "백준 골드 달성 스터디"
- 디자인패턴 + 독서 = "GoF 디자인패턴 완독 스터디"
- 정보처리기사 + 문제 풀이 = "정처기 실기 대비 스터디"
```

### 1.2 주제 테이블 구조

| 필드 | 타입 | 설명 |
|------|------|------|
| id | INT | PK |
| name | VARCHAR(50) | 주제명 |
| parent_id | INT (nullable) | 상위 주제 ID (대분류는 NULL) |

## 2. 형식 (Format)

### 2.1 형식 목록

| 형식 | 설명 | 예시 |
|------|------|------|
| **문제 풀이** | 알고리즘/자격증 문제 풀고 리뷰 | 백준 매일 1문제, 정처기 기출 |

| **모의 면접** | 면접 연습 및 피드백 | 기술면접, 인성면접 |

## 6. DB 스키마 (참고)

```sql
-- 주제 테이블
CREATE TABLE topic (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    parent_id INT NULL,
    icon VARCHAR(50),
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES topic(id)
);

-- 형식 테이블
CREATE TABLE format (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    description VARCHAR(200),
    icon VARCHAR(50),
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 스터디 테이블 (카테고리 관련 필드만)
CREATE TABLE study (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    topic_id INT NOT NULL,
    format_id INT NOT NULL,
    study_type ENUM('PLANNED', 'FLASH') NOT NULL,  -- 계획/번개
    -- ... 기타 필드
    FOREIGN KEY (topic_id) REFERENCES topic(id),
    FOREIGN KEY (format_id) REFERENCES format(id)
);
```

## 7. 확장 고려사항

### 7.1 복수 선택
- 현재: 주제 1개, 형식 1개 선택
- 확장: 주제/형식 복수 선택 (다대다 관계)

### 7.2 커스텀 태그
- 정해진 카테고리 외에 자유 태그 추가 가능
- 예: "SSAFY 11기", "서울캠퍼스", "초보환영"

### 7.3 추천 조합
- 자주 사용되는 주제+형식 조합을 "추천 템플릿"으로 제공
- 예: "알고리즘 문제풀이 스터디 (추천)"
