ID	분류	기능명	요구사항
U-FR-001	회원/인증	회원가입	시스템은 회원가입 시 입력된 이메일로 인증 메일을 발송하고, 인증 완료 후 계정이 활성화되어야 한다.
U-FR-002	회원/인증	이메일 인증	인증 메일은 발송 후 10분 이내에 인증을 완료해야 하며, 만료 시 재발송을 요청할 수 있어야 한다.
U-FR-003	회원/인증	로그인	사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다.
U-FR-004	회원/인증	소셜 로그인	사용자는 네이버, 카카오, 구글 계정으로 간편하게 회원가입 및 로그인할 수 있어야 한다.
U-FR-005	회원/인증	로그아웃	사용자는 언제든지 로그아웃할 수 있어야 한다.
U-FR-006	회원/인증	프로필 설정	사용자는 닉네임, 프로필 이미지, 상태 메시지를 설정할 수 있어야 한다.
U-FR-007	회원/인증	온라인 상태	시스템은 사용자의 온라인/오프라인/자리비움 상태를 실시간으로 표시해야 한다.
U-FR-008	스터디 탐색	스터디 탐색	사용자는 공개된 스터디/팀 목록을 탐색할 수 있어야 한다.
U-FR-009	스터디 탐색	스터디 검색	사용자는 키워드, 카테고리(알고리즘/CS/면접 등), 기술스택으로 스터디를 검색할 수 있어야 한다.
U-FR-010	스터디 탐색	스터디 필터	사용자는 모집중/진행중, 온라인/오프라인/혼합, 지역, 요일, 시간대 등으로 스터디를 필터링할 수 있어야 한다.
U-FR-011	스터디 탐색	스터디 상세 조회	사용자는 스터디의 상세 정보(설명, 규칙, 일정, 현재 멤버 수, 스터디장 평점 등)를 조회할 수 있어야 한다.
U-FR-012	스터디 탐색	스터디 가입 신청	사용자는 관심 있는 스터디에 가입 신청을 할 수 있어야 한다. 신청 시 자기소개를 작성할 수 있다.
U-FR-013	스터디 탐색	신청 관리	스터디장은 가입 신청 목록을 확인하고 승인/거절할 수 있어야 한다.
U-FR-013-1	스터디 탐색	신청 알림	가입 신청이 들어오면 스터디장에게 알림이 발송되어야 한다.
U-FR-013-2	스터디 탐색	수락 알림	스터디장이 신청을 수락하면 신청자에게 "수락되었습니다" 알림이 발송되어야 한다.
U-FR-013-3	스터디 탐색	거절 알림	스터디장이 신청을 거절하면 신청자에게 "거절되었습니다 + 사유" 알림이 발송되어야 한다. 거절 시 사유 입력은 필수이다.
U-FR-013-4	스터디 탐색	재신청 가능	거절된 신청자는 자기소개를 보완하여 다시 신청할 수 있어야 한다.
U-FR-013-5	스터디 탐색	내 신청 목록	사용자는 홈에서 내 스터디 신청 목록을 확인하고 거절 사유 피드백을 받을 수 있어야 한다.
U-FR-014	스터디 탐색	모집글 작성	스터디장은 스터디 모집글을 작성할 수 있어야 한다. (제목, 소개, 모집 인원, 진행 방식, 지역, 일정, 사용 교재/자료, 스터디 목표, 예상 난이도, 사전 지식, 진행 방식 상세)
U-FR-014-1	스터디 탐색	모집글 수정 불가	스터디 모집글은 등록 후 수정할 수 없다. 변경이 필요하면 삭제 후 재생성해야 한다.
U-FR-014-2	스터디 탐색	모집글 삭제 시 알림	스터디장이 모집글을 삭제하고 새로 생성하면, 기존 확정자와 대기중 신청자에게 새 글 알림이 발송된다. (거절된 사람 제외)
U-FR-014-3	스터디 탐색	확정자 자동 승계	삭제 후 재생성 시, 기존 확정자는 새 스터디에 자동으로 확정자로 등록된다.
U-FR-014-4	스터디 탐색	모집글 템플릿 저장	스터디장은 작성한 모집글을 템플릿으로 저장하고 나중에 불러올 수 있어야 한다.
U-FR-014-5	스터디 탐색	기본 템플릿 제공	시스템은 스터디 유형별 기본 템플릿(알고리즘, CS, 면접, 프로젝트, 자격증, 독서/강의)을 제공해야 한다.
U-FR-015	스터디 탐색	모집 상태	스터디장은 모집 상태를 변경할 수 있어야 한다. 모집 종료일에 자동 종료된다.
U-FR-015-1	스터디 탐색	스터디 상태	스터디 상태는 DRAFT(임시저장), SCHEDULED(모집예정), RECRUITING(모집중), RECRUIT_CLOSED(모집완료/시작대기), PENDING(확정대기), IN_PROGRESS(진행중), COMPLETED(완료), CANCELLED(취소)로 구분된다.
U-FR-015-2	스터디 탐색	조기 모집 종료	모집 인원이 충족되면 스터디장은 모집 마감일 전에 조기 마감할 수 있어야 한다.
U-FR-015-3	스터디 탐색	모집 마감 시 자동 거절	모집 기간 종료 시 대기중(승낙도 거절도 안 한) 신청자는 자동으로 거절 처리되며, "모집 마감으로 거절되었습니다" 알림이 발송된다.
U-FR-015-4	스터디 탐색	인원 충족 시 채널 생성	모집 마감 시 인원이 충족되면 정식 채널이 생성되고 스터디 상태가 RECRUIT_CLOSED(시작 대기)가 된다.
U-FR-015-5	스터디 탐색	인원 부족 시 임시 채널	모집 마감 시 인원이 부족하면 임시 채널이 생성되고 스터디 상태가 PENDING(확정 대기)이 된다.
U-FR-015-6	스터디 탐색	임시 채널	임시 채널은 텍스트 채팅만 가능하며, 미팅/자료실/출석 기능은 제공되지 않는다. 확정자들이 진행 여부를 논의하는 공간이다.
U-FR-015-7	스터디 탐색	모집 연장	인원 부족 시 스터디장은 모집 기간을 1회에 한해 연장할 수 있다. 연장 시 스터디 일정도 재등록해야 한다.
U-FR-015-8	스터디 탐색	연장 후 결정	연장 후에도 인원이 부족하면 스터디장은 해당 인원으로 진행하거나 취소를 결정해야 한다. 추가 연장은 불가능하다.
U-FR-015-9	스터디 탐색	정식 채널 전환	스터디장이 진행을 결정하면 임시 채널이 정식 채널로 전환되고, 기존 대화 내역은 삭제된다.
U-FR-015-10	스터디 탐색	스터디 시작일 자동 전환	스터디 시작일이 되면 RECRUIT_CLOSED 상태에서 IN_PROGRESS 상태로 자동 전환되고, 온라인 스터디의 경우 미팅이 자동 시작된다.
U-FR-016	스터디 탐색	모집글 끌어올리기	스터디장은 일정 기간마다 모집글을 상단에 노출시킬 수 있어야 한다.
U-FR-017	스터디 탐색	찜하기	사용자는 관심 있는 스터디를 찜 목록에 저장할 수 있어야 한다.
U-FR-018	스터디 탐색	추천 스터디	시스템은 사용자의 관심사, 기술스택, 일정을 기반으로 맞춤 스터디를 추천해야 한다.
U-FR-019	스터디 탐색	스터디 댓글	사용자는 스터디 모집글에 댓글 및 대댓글을 작성할 수 있어야 한다.
U-FR-019-1	스터디 탐색	스터디 리뷰 작성	스터디 완주자만 해당 스터디에 대한 리뷰(평점 + 텍스트)를 작성할 수 있어야 한다.
U-FR-019-2	스터디 탐색	사용자 프로필 스터디 구분	사용자 프로필에서 "운영한 스터디"와 "참여한 스터디"가 구분되어 표시되어야 한다.
U-FR-019-3	스터디 탐색	스터디장 평점 집계	스터디장 평점은 해당 사용자가 "운영한 스터디"의 리뷰에서만 집계되어야 한다.
U-FR-019-4	스터디 탐색	이전 스터디 조회	스터디 탐색에서 스터디장을 클릭하면 해당 사용자가 운영한 이전 스터디 목록과 리뷰를 조회할 수 있어야 한다.
U-FR-028	멤버/역할	멤버 목록 조회	사용자는 팀에 속한 모든 멤버의 목록과 온라인 상태를 확인할 수 있어야 한다.
U-FR-029	멤버/역할	자동 추방	시스템은 남은 기간 동안 모두 출석해도 50% 이하의 출석률이 되는 멤버를 자동 추방해야 한다.
U-FR-030	멤버/역할	추방 예고 알림	자동 추방 대상자에게 추방 예정일 3일 전 경고 알림을 발송해야 한다.
U-FR-031	멤버/역할	팀장 위임	팀장은 다른 멤버에게 팀장 권한을 위임할 수 있어야 한다.
U-FR-032	친구	친구 검색	사용자는 닉네임으로 다른 사용자를 검색할 수 있어야 한다. (검색 허용 설정한 사용자만)
U-FR-033	친구	친구 요청	사용자는 다른 사용자에게 친구 요청을 보낼 수 있어야 한다.
U-FR-034	친구	친구 수락/거절	사용자는 받은 친구 요청을 수락하거나 거절할 수 있어야 한다.
U-FR-035	친구	친구 목록	사용자는 친구 목록을 조회할 수 있어야 하며, 친구의 온라인 상태가 표시되어야 한다. 서로 맞팔해야 친구 목록에서 볼 수 있다.
U-FR-036	친구	친구 삭제	사용자는 친구를 삭제할 수 있어야 한다.
U-FR-037	친구	사용자 차단	사용자는 다른 사용자를 차단할 수 있어야 한다. 차단된 사용자는 친구 요청, DM을 보낼 수 없다.
U-FR-038	DM	DM 전송	사용자는 친구에게 1:1 다이렉트 메시지를 전송할 수 있어야 한다.
U-FR-039	DM	DM 목록	사용자는 DM 대화 목록을 조회할 수 있어야 한다.
U-FR-040	DM	DM 읽음 표시	DM 메시지의 읽음/안읽음 상태가 표시되어야 한다.
U-FR-041	DM	DM 알림	새 DM 수신 시 알림이 발송되어야 한다.
U-FR-042	채널	채널 생성	팀장/관리자는 텍스트 채널 또는 음성 채널을 생성할 수 있어야 한다.
U-FR-043	채널	채널 카테고리	채널들을 카테고리로 그룹화하여 정리할 수 있어야 한다.
U-FR-044	채널	채널 순서 변경	팀장/관리자는 드래그 앤 드롭으로 채널 순서를 변경할 수 있어야 한다.
U-FR-045	채널	채널 설정	팀장/관리자는 채널 이름, 설명, 접근 권한을 설정할 수 있어야 한다.
U-FR-046	채널	채널 삭제	팀장/관리자는 채널을 삭제할 수 있어야 한다.
U-FR-047	채널	비공개 채널	특정 역할/멤버만 접근 가능한 비공개 채널을 만들 수 있어야 한다.
U-FR-050	텍스트 채팅	메시지 전송	사용자는 텍스트 채널에서 실시간으로 메시지를 전송할 수 있어야 한다.
U-FR-051	텍스트 채팅	메시지 수정	사용자는 자신이 보낸 메시지를 수정할 수 있어야 한다.
U-FR-052	텍스트 채팅	메시지 삭제	사용자는 자신이 보낸 메시지를 삭제할 수 있어야 한다. 관리자는 모든 메시지를 삭제할 수 있다.
U-FR-053	텍스트 채팅	마크다운 지원	메시지에서 마크다운 문법(굵게, 기울임, 코드블록 등)을 지원해야 한다.
U-FR-054	텍스트 채팅	코드 블록	백틱(\`\`\`)으로 감싼 코드는 문법 하이라이팅과 함께 표시되어야 한다.
U-FR-055	텍스트 채팅	파일 첨부	사용자는 이미지, 문서 등 파일을 채팅에 첨부할 수 있어야 한다.
U-FR-056	텍스트 채팅	멘션	사용자는 @닉네임으로 특정 멤버를 멘션할 수 있어야 하며, 멘션된 사용자에게 알림이 가야 한다.
U-FR-057	텍스트 채팅	이모지 반응	사용자는 메시지에 이모지로 반응할 수 있어야 한다.
U-FR-058	텍스트 채팅	메시지 검색	사용자는 채널 내 메시지를 키워드로 검색할 수 있어야 한다.
U-FR-059	텍스트 채팅	메시지 고정	중요한 메시지를 고정하여 쉽게 찾을 수 있어야 한다.
U-FR-060	음성/화상	음성 채널 입장	사용자는 음성 채널에 입장하여 실시간 음성 대화를 할 수 있어야 한다.
U-FR-061	음성/화상	화상 모드	음성 채널에서 카메라를 켜서 화상 미팅으로 전환할 수 있어야 한다.
U-FR-062	음성/화상	화면 공유	사용자는 자신의 화면을 다른 참가자에게 공유할 수 있어야 한다.
