# Squiz 발표 시연 시나리오 (10분)

## 프로젝트 소개 (30초)

> **Squiz**는 "Effortless Learning"을 비전으로 하는 AI 기반 스터디 협업 플랫폼입니다.
>
> **핵심 가치**: 스터디 참여만으로 복습-요약-평가가 자동으로 이루어지는 환경
>
> - **자동화**: 미팅 요약, 회의록, 복습 퀴즈가 AI로 자동 생성
> - **자산화**: 휘발성 대화가 나만의 학습 데이터베이스로 축적
> - **완성도**: 실무 수준의 결과물 (신청서/결과보고서 자동 생성)

---

### 1-2. 스터디 탐색 & 가입 신청 (1분 30초)

**[웹]**

```
시연 순서:
1. 좌측 사이드바 "스터디 탐색" 클릭
2. 스터디 목록 화면
   - 카테고리 필터: 알고리즘/CS/자격증/...
   - 형식 필터: 문제풀이/독서/모의면접/...
   - 상태 필터: 모집중/진행중
3. 관심 스터디 클릭 → 상세 화면
   - 스터디 설명, 규칙, 일정
   - 스터디장 평점 / 이전 스터디 리뷰
   - 현재 멤버 수 / 모집 인원
4. "가입 신청" 버튼 클릭
5. 자기소개 입력
6. 신청 완료 → "내 신청 목록"에서 확인
```

**포인트 멘트:**
> "주제와 형식으로 원하는 스터디를 쉽게 찾을 수 있습니다. 스터디장의 이전 운영 이력과 리뷰를 확인하고 신청할 수 있습니다."

## Part 2. 채널 채팅 & 실시간 소통 (2분)

### 2-1. 텍스트 채널 (1분)

**[웹]**

```
시연 순서:
1. 가입된 스터디 선택
2. 좌측 채널 목록 확인
   - 📁 공지사항
   - 📁 자유채팅
   - 📁 코드리뷰
   - 🔊 음성채널
3. "#자유채팅" 채널 입장
4. 메시지 입력 (마크다운)
   - **굵게**, *기울임*
   - 코드블록: ```python print("Hello")```
5. @멘션 기능 시연
6. 파일 첨부 (이미지/문서)
7. 이모지 반응 추가
```

**포인트 멘트:**
> "Discord와 유사한 채널 기반 구조입니다. 마크다운, 코드블록을 지원하며 @멘션 시 해당 멤버에게 알림이 발송됩니다."

### 2-2. 실시간 동기화 (1분)

**[웹 2개 창]**

```
시연 순서:
1. 브라우저 2개 창으로 다른 계정 로그인
2. 같은 채널에서 메시지 송수신
3. 실시간으로 메시지가 동기화되는 것 확인
4. 한쪽에서 메시지 수정 → 다른 쪽에서 반영 확인

```

**포인트 멘트:**

## Part 3. 화상 미팅 & AI 요약 (2분 30초)

### 3-1. 화상 미팅 시작 (1분)

**[웹]**

```
시연 순서:
1. 🔊 음성채널 클릭 → "입장" 버튼
2. 미팅 화면 진입
   - 참가자 영상 그리드
   - 하단 컨트롤: 🎤 마이크 / 📷 카메라 / 🖥️ 화면공유
3. 마이크/카메라 ON/OFF 시연
4. 화면 공유 시연
5. 발언자 하이라이트 표시 확인
6. "미팅 시작" 버튼 클릭 → 녹음 시작
```

**포인트 멘트:**
> "mediasoup SFU 서버를 통해 다자간 화상 미팅을 지원합니다. '미팅 시작'을 누르면 녹음이 시작되어 AI 요약에 사용됩니다."

---

### 3-2. 미팅 종료 & AI 요약 (1분 30초)

**[웹]**

```
시연 순서:
1. 미팅에서 간단한 대화 진행 (1-2분)
   - "오늘은 DP 문제를 복습하겠습니다"
   - "백준 1463번 풀이 방법을 공유할게요"
2. "미팅 종료" 버튼 클릭
3. "AI 요약 생성 중..." 로딩
4. 미팅 기록 화면으로 이동
   - [요약 탭] 전체 미팅 요약
   - [전문 탭] STT 변환 결과
   - [액션아이템 탭] 할 일 목록
5. 액션아이템에 담당자 지정
```

**포인트 멘트:**
> "미팅이 종료되면 Whisper로 음성을 텍스트로 변환하고, 자체 파인튜닝한 Qwen3-8B가 1차 요약을 생성합니다. 이후 Claude가 검토/보완하여 높은 품질의 최종 결과물을 제공합니다."

## Part 4. BLE 출석 체크 (1분 30초)

**[모바일 앱 2대]**

```
시연 순서:
1. [스터디장 폰] 스터디 관리 → "출석 시작" 버튼
   - BLE 비콘 신호 발신 시작
   - "출석 대기 중..." 화면 표시
2. [스터디원 폰] 앱 실행
   - BLE 신호 감지
   - "출석 완료!" 알림
3. [스터디장 폰] 실시간 출석 현황 확인
   - ✅ 홍길동 - 출석
   - ✅ 김철수 - 출석
   - ⏳ 이영희 - 대기중
4. 출석 캘린더에서 기록 확인
```

**포인트 멘트:**
> "오프라인 스터디에서는 스터디장이 BLE 비콘을 발신하면 주변 스터디원의 앱이 자동으로 출석을 체크합니다. 별도 장비 없이 스마트폰만으로 가능합니다."

## Part 5. 퀴즈 시스템 (1분)


> "카테고리별 퀴즈 대회를 운영할 수 있습니다. 실시간으로 순위가 업데이트되고, TOP3는 퀴즈왕 뱃지를 받습니다."

## 추가 시연 가능 기능 (시간 여유 시)

1. **스터디 모집글 작성**: 템플릿 선택, 주제/형식 설정
2. **신청 관리**: 가입 신청 승인/거절, 알림 발송
3. **데일리 스탠드업**: 어제 한 일 / 오늘 할 일 / 블로커
4. **회고 작성**: KPT 템플릿, 투표 기능
5. **자료실**: 학습 자료 공유, 태그/폴더 정리
6. **커리큘럼**: 주차별 진도 관리
7. **DM**: 친구와 1:1 메시지
8. **알림 설정**: 알림 유형별 ON/OFF
9. **복습 퀴즈**: AI 생성 퀴즈 풀이
10. **꼬멘틀 게임**: IT 용어 추측 (Wordle 스타일)

## 시연 시 유의사항

1. **네트워크 확인**: 데모 전 WiFi 연결 상태 확인
2. **테스트 데이터**: 충분한 스터디/메시지/미팅 기록 준비
3. **백업 영상**: 네트워크 문제 대비 녹화 영상 준비
4. **음성 테스트**: 미팅 시연 전 마이크/스피커 테스트
5. **BLE 권한**: 모바일 앱 블루투스/위치 권한 미리 허용
