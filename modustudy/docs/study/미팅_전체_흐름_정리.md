# ModuStudy 미팅 기능 전체 흐름 정리

## 1. 목적
- 스터디 내 공식 미팅(음성/화상)을 시작하고, 기록/요약/액션아이템까지 자동화한다.
- 실시간 화상/음성 + 채팅 + 녹화/녹음 + STT/요약 + 과거 기록 조회/내보내기를 한 흐름으로 연결한다.

## 2. 주요 구성 요소
- Frontend (meeting 모듈)
  - 미팅 목록/상세/룸 화면
  - STOMP(WebSocket) 채팅/참가자/발표자 상태
  - SFU(WebRTC) 미디어 송수신
  - 캔버스 합성(화면 공유 + 캠 PIP)
  - 녹화/녹음 업로드
- Backend (meeting 도메인)
  - 미팅 생성/종료, 기록, 요약, 액션아이템, 내보내기 API
  - 미팅 웹소켓 이벤트/채팅 저장
  - SFU 설정 제공 API
- SFU 서버
  - mediasoup 기반 미디어 라우팅
- STT/AI 요약 파이프라인
  - 녹음 파일 업로드 후 STT 및 요약 처리 (서버 측)

## 3. 화면/라우팅 흐름
- 미팅 목록: /study/:studyId/meetings
- 미팅 상세: /study/:studyId/meetings/:meetingId
- 미팅 룸: /study/:studyId/meetings/:meetingId/room

## 4. 미팅 시작 흐름
1) 사용자: '미팅 시작' 버튼 클릭
2) 모달에서 유형 선택(데일리/주간/자유/기타) 및 자동 공유 옵션 지정
3) Backend: POST /api/v1/studies/{studyId}/meetings
   - 미팅 생성, 상태 WAITING -> IN_PROGRESS
4) Frontend: 룸 화면으로 이동

## 5. 룸 입장/연결 흐름
1) Backend: POST /api/v1/studies/{studyId}/meetings/{meetingId}/join
2) WebSocket(STOMP)
   - /ws 연결
   - /app/rooms/{roomId}/join로 참가자 등록
   - /topic/rooms/{roomId}/events 로 참가자/발표자/채팅 이벤트 수신
3) SFU
   - /api/v1/sfu/config 로 SFU baseUrl/ICE 서버 정보 획득
   - socket.io로 SFU 룸 접속
   - mediasoup transport 생성 후 audio/video producer 생성

## 6. 발표자 제어 및 미디어 정책
- 발표자는 한 회의에 1명만 가능
- 발표자만 카메라/화면공유 가능
- 발표자 변경 시 확인 모달 후 기존 발표자 권한 해제, 신규 발표자 권한 부여
- 발표자가 화면 공유 + 캠 동시 사용 시:
  - Canvas 합성으로 하나의 스트림 생성
  - 캠 위치(좌상/우상/좌하/우하) 선택 가능

## 7. 채팅/참가자 상태
- 실시간 채팅
  - /app/rooms/{roomId}/chat 전송
  - 서버에서 저장 후 이벤트 전파
- 참가자 목록
  - 말하는 사람 표시(오디오 감지 기반)
  - 자리에 없음/있음 표시(AI 사람 감지 기반)

## 8. 녹화/녹음/저장 흐름
- 미팅 시작 시 녹화/녹음 자동 시작
- 종료 시 자동 저장 및 업로드
- 업로드 대상
  - 전체 녹화(video)
  - 개인 음성(INDIVIDUAL, userId 포함)
  - 전체 음성(MIXED)
- 관련 API
  - POST /api/v1/studies/{studyId}/meetings/{meetingId}/recording/video
  - POST /api/v1/studies/{studyId}/meetings/{meetingId}/recording/audio

## 9. STT/요약/액션아이템
- 미팅 종료 시
  - STT 변환(발언자 구분)
  - 전체 요약 + 개인별 요약 생성
  - 액션아이템 자동 추출
- 결과 저장
  - STT 텍스트 파일 업로드: /stt/file
  - 요약 텍스트 파일 업로드: /summary/file
  - 액션아이템 관리: /action-items

## 10. 미팅 종료/기록 조회
- 종료 버튼
  - PUT /api/v1/studies/{studyId}/meetings/{meetingId}/end
- 기록 조회
  - 목록: GET /api/v1/studies/{studyId}/meetings
  - 상세: GET /api/v1/studies/{studyId}/meetings/{meetingId}
  - 채팅: GET /api/v1/studies/{studyId}/meetings/{meetingId}/chat

## 11. 내보내기
- Markdown/PDF 내보내기
  - GET /api/v1/studies/{studyId}/meetings/{meetingId}/export?format=MARKDOWN|PDF

## 12. 환경 설정
- Frontend .env
  - VITE_API_URL=http://localhost:8080
- WebSocket
  - /ws STOMP 엔드포인트
