## 3. 경쟁 서비스 대비 상세 차별점

### TOP 1. Commentle (꼬멘틀)

**한줄 설명**: 매일 출제되는 CS 용어를 AI 임베딩 유사도를 기반으로 추론하며, 시도한 단어들의 의미적 거리가 3D 공간에서 실시간 시각화되는 게임형 학습 도구.

| 경쟁 서비스 | 한계 | SQUIZ 차별점 |
|---|---|---|
| 프로그래머스 | 단순 문제풀이, 정답/오답 이진 피드백 | AI가 유사도 점수를 0~100으로 제공, 연속적 피드백 |
| 인프런 | 수동적 강의 시청 | 능동적 추론 과정을 통한 개념 간 관계 학습 |
| Wordle | 영단어 철자 맞추기, 학습 목적 아님 | CS 도메인 특화, S-ROBERTA 모델로 의미적 유사도 계산 |

**사용 시나리오**: 개발자 취준생이 매일 아침 Commentle을 풀며 "해시테이블"이라는 정답을 맞추기 위해 "배열" → 32점, "딕셔너리" → 78점, "해시맵" → 95점으로 점점 접근. 3D 공간에서 자신의 시도들이 정답에 가까워지는 것을 시각적으로 확인하면서, 자연스럽게 CS 용어 간의 의미적 관계를 체화.

### TOP 2. STT 기반 AI 미팅 요약

**한줄 설명**: 화상 미팅 종료 후 음성 인식(STT)을 통해 대화 내용을 자동 텍스트화하고, AI가 요약/키워드/액션 아이템/화자별 통계를 자동 생성.

| 경쟁 서비스 | 한계 | SQUIZ 차별점 |
|---|---|---|
| Zoom | 유료 AI Companion, 일반 회의용 | 스터디 맥락에 특화된 요약, 학습 액션 아이템 자동 추출 |
| Google Meet | 기본적인 자막만 제공 | 요약/키워드/하이라이트/액션아이템/화자별 통계 5가지 인사이트 |
| Discord | 녹음/요약 기능 없음 | 미팅 후 대시보드에서 리포트 위젯으로 즉시 복습 가능 |

**사용 시나리오**: 알고리즘 스터디에서 1시간 미팅 후, 참석하지 못한 멤버가 대시보드의 STT 리포트 위젯에서 요약을 확인. "다음 주까지 DP 문제 5개 풀기"라는 액션 아이템이 자동 추출.

### TOP 3. mediasoup SFU 화상회의

**한줄 설명**: 전용 SFU 서버를 통해 P2P 한계를 넘어선 다자간 화상 통화와, OffscreenCanvas + Web Worker를 활용한 백그라운드 안정 녹화.

| 경쟁 서비스 | 한계 | SQUIZ 차별점 |
|---|---|---|
| Zoom | 외부 서비스 의존, 스터디 컨텍스트 단절 | 워크스페이스 내 원클릭 미팅 진입, 스터디 세션과 자동 연동 |

## 4. 기술적 임팩트 포인트

### AI 통합이 만드는 UX 혁신

4가지 AI 기술을 단일 플랫폼에 통합하여, 학습의 전 과정(계획-실행-복습-분석)을 자동화:

1. **GPT 스트리밍 커리큘럼 생성** → 스터디 생성의 진입장벽 제거
2. **S-ROBERTA 의미 임베딩** → "얼마나 가까운지" 연속적 스펙트럼 피드백
3. **STT + AI 요약** → 미팅 종료 후 2~10분 내 리포트 자동 생성
4. **간격 반복 알고리즘** → 망각 곡선에 따라 최적 복습 시점에 재출제

**핵심**: 4가지가 개별 기능이 아니라 **학습 사이클로 연결** — AI가 커리큘럼을 짜고(계획) → 화상 미팅으로 학습하고(실행) → STT가 미팅을 요약하고(기록) → 퀴즈로 복습하고(강화) → 약점 분석으로 다음 학습을 안내(개선)

### 실시간 통신 (WebRTC + WebSocket)

- **mediasoup SFU**: 서버 중계 방식으로 클라이언트 부하 최소화. Simulcast 2레이어 인코딩으로 적응적 품질 제어
- **STOMP WebSocket**: JOIN/LEAVE/MESSAGE/PIN/DELETE/PRESENCE 6종 이벤트 실시간 처리. 3분 미활동 시 IDLE 자동 전환. REST API 자동 폴백

### 게이미피케이션이 학습 지속에 미치는 영향

1. **즉각적 보상**: 모든 학습 활동에 경험치 부여
2. **축하 의식**: Confetti + 돌고래 애니메이션 풀스크린 레벨업 모달
3. **시각적 성장 추적**: GitHub 잔디를 레고 블록 스타일로 재해석한 활동 히트맵

## 5. PPT 구성안 (18장)

### 슬라이드 1. 타이틀

- **제목**: SQUIZ — Study + Quiz, 하나의 공간에서
- **핵심 문구**: "흩어진 학습을, 하나로 모으다"
- **시각 자료**: 로고 중앙 + 대시보드 스크린샷 배경 반투명. 하단에 팀명, 발표자, 날짜
- **멘트**: "오늘 소개드릴 서비스는 SQUIZ입니다. 온라인 스터디의 모든 과정을 하나의 플랫폼 안에서 완결시키는 올인원 학습 협업 서비스입니다."

### 슬라이드 2. 문제 제기

- **제목**: 온라인 스터디, 정말 잘 되고 있나요?
- **핵심**: 분산 / 이탈 / 비효율
- **시각 자료**: Zoom, 카톡, 노션, 구글드라이브 등 6~7개 앱 아이콘이 흩어진 인포그래픽. "스터디 중도 이탈률 67%", "평균 사용 툴 4.3개"
- **멘트**: "화상회의는 Zoom, 자료 공유는 구글드라이브, 소통은 카톡, 일정은 노션... 도구가 흩어지면 학습에 집중하기 어렵습니다."

### 슬라이드 3. Pain Point 3가지

- **제목**: 스터디가 무너지는 3가지 이유
- **내용**:
  1. "도구가 흩어져 있어 맥락이 끊긴다"
  2. "출석, 진도 관리가 안 되어 긴장감이 사라진다"
  3. "학습 성과를 체감할 수 없어 동기가 떨어진다"
- **시각 자료**: 끊어진 사슬, 빈 체크리스트, 하락 그래프 아이콘
- **멘트**: "이 세 가지가 해결되지 않으면, 스터디는 반드시 무너집니다."

### 슬라이드 4. 솔루션 선언

- **제목**: SQUIZ가 답입니다
- **핵심 문구**: "회의 - 학습 - 관리 - 동기부여, 하나의 플랫폼에서 끝"
- **시각 자료**: 중앙 로고 + 4개 키워드 원형 연결 다이어그램. Before/After 느낌
- **멘트**: "더 이상 여러 도구 사이를 오가실 필요가 없습니다."

### 슬라이드 5. 전체 구조 한눈에

- **제목**: SQUIZ 한눈에 보기
- **시각 자료**: 대시보드 중심 방사형 6개 핵심 모듈 맵 — 스터디 관리, 화상회의, 워크스페이스, SQUIZ 아일랜드, 캘린더, 자료실
- **멘트**: "대시보드를 중심으로 6개 핵심 모듈이 유기적으로 연결되어 있습니다."

### 슬라이드 6. 핵심 기능 - 대시보드

- **제목**: 맞춤형 대시보드 - 내 학습의 컨트롤 타워
- **핵심 문구**: "로그인하는 순간, 오늘 해야 할 것이 보입니다"
- **시각 자료**: 대시보드 스크린샷 (복습 퀴즈, 스터디 현황, STT 리포트, 목표 위젯)
- **멘트**: "'오늘 뭘 해야 하지?'라는 고민 없이, 바로 학습에 돌입할 수 있습니다."

### 슬라이드 7. 핵심 기능 - AI 커리큘럼

- **제목**: AI가 설계하는 스터디 커리큘럼
- **핵심 문구**: "주제만 입력하면, GPT가 주차별 커리큘럼을 실시간으로 생성합니다"
- **시각 자료**: 스터디 생성 폼 → GPT 스트리밍 생성 화면 시퀀스
- **멘트**: "'자바 스프링 백엔드'라고 입력하면 GPT가 실시간으로 8주 커리큘럼을 생성합니다. 스터디 리더가 가장 어려워하는 '뭘 공부할까' 문제를 AI가 해결합니다."

### 슬라이드 8. 핵심 기능 - 화상회의 + AI 요약

- **제목**: 회의도 하고, 기록도 남기고
- **핵심 문구**: "만나서 끝이 아닙니다. AI가 회의록을 써줍니다"
- **시각 자료**: 화상회의 화면 + 하단 STT 요약 결과 (요약문 + 액션 아이템)
- **멘트**: "회의가 끝나면 AI가 음성을 텍스트로 변환하고, 핵심 내용을 요약하고, 액션 아이템까지 추출합니다. 회의에만 집중하세요, 기록은 AI가 합니다."

### 슬라이드 9. 핵심 기능 - SQUIZ 아일랜드

- **제목**: SQUIZ 아일랜드 - 학습을 게임처럼
- **핵심 문구**: "외우는 게 아니라, 풀면서 체화합니다"
- **시각 자료**: Three.js 3D Commentle 화면 + Practice Mode UI + 간격 반복 곡선 그래프
- **멘트**: "S-ROBERTA 임베딩 모델로 CS 용어의 의미적 유사도를 계산하는 추론 게임입니다. Practice Mode는 간격 반복 알고리즘을 적용해서, 틀린 문제가 최적의 타이밍에 다시 출제됩니다."

### 슬라이드 10. 핵심 기능 - 워크스페이스

- **제목**: 슬랙 같은 실시간 소통 공간
- **핵심 문구**: "스터디 안에서 바로 소통, 바로 공유"
- **시각 자료**: 워크스페이스 채팅 UI (메시지 핀, 파일 공유, 멤버 리스트)
- **멘트**: "실시간 채팅, 파일 공유, 핀 메시지, 캘린더와 자료실까지 한 곳에서 접근. 대화와 학습 자료가 분리되지 않으니 맥락이 절대 끊기지 않습니다."

## 8. Q&A 대비 예상 질문 + 답변

### Q1. "기존 노션+Zoom 조합 대비 전환 가치가 있나요?"

> "핵심은 '맥락의 연결'입니다. Zoom에서 나눈 이야기가 노션에 자동으로 정리되지 않고, 노션의 일정이 Zoom과 연동되지 않습니다. SQUIZ는 회의에서 나온 액션 아이템이 자동 정리되고, 퀴즈에서 틀린 문제가 대시보드에 복습 알림으로 뜹니다. 이 연결성이 개별 도구 조합으로는 불가능한 가치입니다."

### Q2. "mediasoup SFU를 선택한 이유?"

> "Janus는 C 기반이라 커스터마이징이 어렵고, Kurento는 유지보수가 활발하지 않습니다. mediasoup은 Node.js 기반으로 프론트엔드 팀과 기술 스택을 공유할 수 있고, Worker 단위 스케일링이 가능해 확장성에서 유리합니다."

### Q3. "동시 접속 몇 명까지?"

> "화상회의는 한 방 6~8명 기준 최적화. SFU Worker 추가로 수평 확장 가능합니다. 채팅은 WebSocket + Redis pub/sub으로 다중 서버 환경도 지원합니다."

### Q4. "게이미피케이션 효과 근거?"

> "Duolingo가 스트릭/경험치/리그 시스템으로 일일 학습 지속률을 6.7배 향상시켰다는 데이터를 공개한 바 있습니다. GitHub 잔디도 같은 원리입니다. '빈칸을 채우고 싶은' 심리가 지속적인 활동을 유도합니다."

### Q5. "보안 처리?"

> "JWT + OAuth 2.0 (Google/Kakao/Naver) + Spring Security 6 API 엔드포인트별 권한 관리. WebRTC 통신은 DTLS/SRTP 암호화. 파일 업로드 시 확장자 검증 및 용량 제한 적용."

### Q6. "S-ROBERTA 임베딩 정확도?"

> "문장 수준 의미적 유사도 측정에 최적화된 모델입니다. CS 용어 데이터셋으로 유사도 임계값을 튜닝했고, 'HTTP'와 'REST API', 'Array'와 'LinkedList' 등의 유사도가 직관적으로 납득 가능한 수준입니다."

### Q7. "프로젝트 기간과 팀 구성?"

> "팀원 [N]명이 약 [기간] 동안 개발했습니다. 프론트엔드 [N]명, 백엔드 [N]명, 인프라/AI [N]명으로 역할 분담. 2주 단위 스프린트, Git Flow + 코드 리뷰로 품질 관리."

3. **숫자로 말하라** — "많은 기능" 대신 "6개 핵심 모듈", "여러 도구" 대신 "평균 4.3개 앱"
