# SQUIZ 프로젝트 발표 가이드 (Version 3 - 일반인 대상)

> 대상: 비개발자, 일반 청중, 투자자, 서비스 기획자 등
> 톤: 기술 용어 최소화, "왜 써야 하는지"에 집중, 체감 가능한 가치 위주

---

## 1. 서비스 한 문장 정의

> **"스터디 모임을 만들고, 만나고, 복습하는 — 스터디에 필요한 모든 것을 하나로 통합한 AI 학습 플랫폼"**

**보조 정의 (상황별)**:
- 엘리베이터 피치용: "카톡으로 일정 잡고, 줌으로 만나고, 노션에 기록하던 걸 — 하나의 앱에서 AI가 도와줍니다."

---

## 2. 핵심 가치 3가지

| 가치 | 설명 |
|------|------|
| **AI가 미팅을 학습으로 바꿈** | 평범한 미팅이 끝나면 AI가 자동 요약, 기록 문서 생성, 복습 퀴즈 출제, 화자 분리까지 — 만나기만 하면 학습 자산이 만들어짐 |
| **학습 데이터 자산화** | 풀었던 퀴즈, 틀린 문제, 약점 분석, 모임 기록이 쌓여 나만의 학습 데이터가 됨. 시간이 지날수록 AI가 나를 더 잘 이해하고 맞춤 복습을 제공 |
| **노력 없이 학습되는 환경** | 출석은 자동, 복습은 AI가 알림, 계획은 AI가 생성 — 사용자는 그냥 참여하기만 하면 자연스럽게 학습이 이루어지는 환경 |

## 3. 핵심 기능 정리 (일반인 관점)

### 3-1. AI가 알아서 만들어주는 학습 계획

**이전**: 스터디 리더가 혼자 고민하며 몇 시간 걸려 커리큘럼 작성
**SQUIZ**: 주제만 입력하면 AI가 몇 초 만에 주차별 학습 계획을 만들어줌

- "자바 백엔드" 입력 → 8주 커리큘럼 자동 생성
- 마음에 안 들면 자유롭게 수정 가능
- 생성된 커리큘럼은 도로 모양 로드맵으로 시각화 (빨간 자동차가 현재 위치 표시)

> **임팩트 멘트**: "뭘 공부할지, 계획은 어떻게 세울지 고민하느라  공부를 못 시작하신 적 있으시죠? 주제만 알려주세요. AI가 계획을 도와드립니다."

---

### 3-2. 화상 모임 + AI 자동 회의록

**이전**: 줌으로 모이고, 누군가 회의록을 수동으로 작성
**SQUIZ**: 화상 모임이 끝나면 AI가 자동으로 요약해줌

- 여러 명이 동시에 화상 모임 가능
- 카메라/마이크/화면 공유 지원
- 모임이 끝나면:
  - 대화 내용 자동 텍스트 변환
  - 핵심 내용 요약
  - "다음에 할 일" 목록 자동 추출
  - 누가 얼마나 말했는지 화자별 비율 통계까지 제공

> **임팩트 멘트**: 
"회의록 담당자를 정하지 않아도 됩니다. AI가 대신 정리해줍니다."
"개인적으로 귀찮은데 시간내서 노션에 정리할 필요 없습니다."

---

### 3-3. 퀴즈로 확실하게 복습 (SQUIZ)

**이전**: 공부한 내용을 따로 복습하지 않아 금방 잊어버림
**SQUIZ**: AI가 최적의 타이밍에 복습 문제를 출제

#### 매일 복습 퀴즈 (핵심)
- "오늘 복습해야 할 문제"가 매일 대시보드에 표시
- 틀린 문제는 더 자주, 맞힌 문제는 덜 자주 나타남
- 과학적으로 검증된 **망각 곡선** 원리 적용 — 잊어버리기 직전에 다시 복습
- 약점 분석: 어떤 분야가 부족한지 한눈에 파악

#### 코스별 학습
- 운영체제, 네트워크, 데이터베이스 등 분야별 퀴즈 코스
- 단계적으로 학습하며 진행률 확인
- 코스 완료 시 해당 코스마스터 뱃지 획득

#### 단어 추론 게임 (꼬멘틀) => 브레인스토밍을 통해 학습유도
- 매일 출제되는 단어를 힌트를 보고 추론하는 게임
- 입력한 단어가 정답과 얼마나 가까운지 점수로 표시
- 3D 화면에서 시각적으로 확인 가능
- 리더보드에서 다른 사용자와 경쟁

> **임팩트 멘트**: "한 번 틀린 문제는 잊어버릴 때쯤 다시 나타납니다. 과학적으로 증명된 복습법을 AI가 자동으로 실행합니다."

---

### 3-4. 디스코드처럼 소통하는 워크스페이스

**이전**: 카카오톡에서 자료가 묻히고, 중요한 메시지를 놓침
**SQUIZ**: 스터디 전용 소통 공간

- 실시간 채팅
- 중요 메시지 핀 고정
- 파일 공유 + 자료실 연동
- 멤버 접속 상태 확인 (온라인/자리비움/오프라인)
- 같은 화면에서 캘린더, 자료실, 모임 기록까지 접근

> **임팩트 멘트**: "카톡에서 자료 찾느라 스크롤 올리신 적 있으시죠? 더 이상 그럴 필요 없습니다."

---

### 3-5. 통합 캘린더 + Google Calendar 연동

**이전**: 개인 일정과 스터디 일정을 따로 관리
**SQUIZ**: 한 화면에서 모든 일정 확인

- 내 개인 일정 + 스터디 모임 일정이 한 캘린더에 표시
- Google Calendar과 자동 연동 — 스터디 일정이 구글 캘린더에도 뜸
- 일정 충돌 한눈에 파악
- 빠른 일정 추가

---

### 3-6. 체계적인 스터디 관리

- **스터디 만들기**: 일반 스터디 / 번개 스터디 선택
- **지원자 관리**: 신청 승인/거절
- **출석 관리**: 출석/지각/결석 기록, 통계 확인
- **멤버 관리**: 역할 변경, 탈퇴 처리

> **임팩트 멘트**: "스터디장이 카톡으로 일일이 출석 확인하던 시대는 끝났습니다."

---

### 3-7. 성장이 보이는 게이미피케이션

- **레벨 시스템**: 활동할수록 경험치가 쌓이고 레벨이 오름
- **뱃지 수집**: 특정 목표 달성 시 뱃지 획득
- **레벨업 축하**: 레벨업하면 화면에 축하 애니메이션 (돌고래가 축하해줌!)
- **활동 기록**: GitHub 잔디처럼 내 학습 기록이 시각적으로 쌓임

> **임팩트 멘트**: "매일 조금씩 쌓이는 학습 기록. 한 달 뒤 돌아보면, 내가 얼마나 성장했는지 눈에 보입니다."

## 4. 모바일 앱 (Android)

### 앱으로 더 편리하게

SQUIZ는 웹뿐만 아니라 **Android 앱**으로도 이용할 수 있습니다. 앱에서만 가능한 특별한 기능들이 있습니다.

### 4-1. 블루투스 출석 체크 (앱 전용 핵심 기능)

**이전**: "누가 왔어요?" 카톡으로 일일이 확인
**SQUIZ 앱**: 스터디장 근처에 가면 자동으로 출석 완료

- **스터디장**: 앱에서 "출석 시작" 버튼만 누르면 됨
- **스터디원**: 스터디장 근처(약 10m 이내)에 있으면 자동으로 출석 인식
- 블루투스(BLE) 기술로 실내에서도 정확하게 작동
- GPS보다 정확하고, 배터리 소모가 적음
- 실시간으로 누가 출석했는지 스터디장 화면에 표시

> **임팩트 멘트**: "오프라인 스터디에서 출석 체크하려고 명단 돌릴 필요 없습니다. 폰만 가지고 있으면 자동으로 체크됩니다."

### 4-2. 푸시 알림

- 스터디 모임 시작 전 알림
- 새 채팅 메시지 알림
- 출석 체크 시작 알림
- 복습 퀴즈 리마인드 알림

> 중요한 일정을 절대 놓치지 않습니다.

### 4-3. 모임 녹음

- 오프라인 스터디 중 앱으로 녹음 가능
- 백그라운드에서도 녹음 유지 (다른 앱 사용 가능)
- 녹음 파일 자동 저장

### 4-4. 앱 주요 화면 구성

| 탭 | 기능 |
|----|------|
| **홈** | 내 스터디 현황, 오늘의 복습, 다가오는 일정 |
| **스터디** | 스터디 탐색/관리, 채팅, 자료실 |
| **퀴즈** | 매일 복습, 코스 학습, 퀴즈 대회, 오답 노트 |
| **마이페이지** | 프로필, 레벨/뱃지, 활동 기록, 설정 |

### 4-5. 웹 vs 앱 비교

| 기능 | 웹 | 앱 |
|------|:---:|:---:|
| 스터디 관리 | O | O |
| 화상 모임 | O | - |
| 채팅 | O | O |
| 퀴즈/복습 | O | O |
| 캘린더 | O | O |
| **블루투스 출석** | - | **O** |
| **푸시 알림** | - | **O** |
| **모임 녹음** | - | **O** |
| AI 커리큘럼 | O | - |
| AI 회의 요약 | O | - |

> 웹과 앱이 서로 보완하여 **어디서든, 어떤 상황에서든** 스터디를 이어갈 수 있습니다.

### 슬라이드 2. 공감 유도 - 이런 경험 있으시죠?

#### 레이아웃
- **상단**: 제목 (큰 폰트, 볼드)
- **중앙**: 핵심 텍스트가 주인공 — 텍스트를 크고 명확하게 배치
- **중앙 좌측**: 온라인 스터디 불편 텍스트 블록
- **중앙 우측**: 오프라인 스터디 불편 텍스트 블록
- **하단 보조**: 각 서비스 공식 로고 아이콘 (카카오톡, Zoom, Notion, Google Drive) — 작게 배치하되 텍스트를 압도하지 않도록

#### 페이지 텍스트
- **제목**: "스터디, 이렇게 하고 계시진 않나요?"
- **온라인 스터디**:
  - "일정은 카톡으로"
  - "모임은 줌으로"
  - "기록은 노션에"
  - "자료는 구글드라이브에"
- **오프라인 스터디**:
  - "출석은 카톡으로 일일이 확인"
  - "회의록은 누군가 손으로 정리"
  - "복습은 각자 알아서"
  - "녹음 파일은 폰에 묻힘"
- **하단 보조 아이콘**: 카카오톡 / Zoom / Notion / Google Drive 공식 로고 (소형)

> **디자인 지시**: 텍스트가 슬라이드의 중심이 되어야 함. 로고 아이콘은 보조 역할로만 사용하고, 텍스트보다 시각적으로 튀지 않도록 크기와 투명도 조절.

#### 멘트
> "여러분, 스터디 해보신 적 있으시죠? 온라인이든 오프라인이든 상황은 비슷합니다. 카톡으로 일정 잡고, 줌으로 만나고, 노션에 정리하고... 오프라인이면 출석을 일일이 체크하고, 누군가 회의록을 손으로 써야 하죠. 도구가 이렇게 흩어져 있으면, 공부보다 관리가 더 힘들어집니다."

### 슬라이드 3. 문제 정의 - 스터디가 무너지는 이유

#### 레이아웃
- **상단**: 제목 (큰 폰트, 볼드)
- **좌측 (40%)**: 이탈 타임라인 그래프 — 주차별 참여율 하락 곡선
- **우측 (60%)**: 실패 패턴 5가지 — 카드형 또는 번호 리스트, 텍스트 중심으로 크게
- **하단**: 한 줄 결론 문구 (강조색 배경)

> **디자인 지시**: 좌측 그래프는 직관적인 하락 곡선으로 "꺾이는 지점"을 시각적으로 보여주되, 우측 텍스트가 핵심 메시지이므로 비중을 더 크게 잡을 것.

#### 페이지 텍스트
- **제목**: "왜 스터디는 5주를 못 넘길까?"
- **좌측 — 이탈 타임라인** (Y축: 참여율%, 하락 곡선 그래프):
  - 1~4주차: "초기 동기, 비교적 유지" **(참여율 70~90%)**
  - 5~6주차: "가장 많이 꺾이는 전환점" **(초기 대비 20~30%p 하락)** (강조 표시, 곡선 기울기 가장 가파른 구간)
  - 8주차 이후: "계획 인원 vs 실제 참여 괴리" **(누적 이탈률 15~20% 이상, 실참여 절반 이하)**
  - 그래프 포인트 참고: 1주차 90% → 3주차 80% → 5주차 60% → 8주차 50%
- **우측 — 실패하는 그룹의 전형적 특징**:
  1. "사전 준비 부족" — 누군가 읽어주고, 나머지는 구경 **(과제 제출률 50% 미만이 2주 이상 지속)**
  2. "소수에 집중된 부담" — 스터디장 혼자 다 떠안는 구조 **(발표·정리를 3회 연속 같은 사람이 수행)**
  3. "역할·아젠다 부재" — 대충 모여서 대충 공부 **(회차 전날까지 아젠다 공유 없는 비율 30% 이상)**
  4. "목표·속도 불일치" — 각자 기대 수준이 달라 불만 누적 **(만족도 5점 중 평균 3점 미만)**

  - X축: 시간 (1일 → 3일 → 7일 → 14일 → 30일)

  - 곡선 1 (회색, 점선): "복습 안 했을 때" — 급격히 하락
  - 곡선 2 (강조색, 실선): "SQUIZ 복습 적용" — 각 복습 시점에서 회복, 점점 높은 수준 유지

> - 망각 곡선 그래프가 핵심 — 두 곡선의 차이가 극적으로 보이도록 색상 대비 강하게
> - 우측 스크린샷은 그래프를 보완하는 역할, 크기 과하지 않게

## 8. Q&A 대비 (일반인 질문 위주)

### Q1. "기존에 줌+카톡+노션 조합으로 잘 되는데, 왜 이걸 써야 하나요?"

> "각각은 좋은 도구입니다. 하지만 문제는 연결이 안 된다는 거예요. 줌에서 한 이야기가 노션에 자동으로 안 가고, 카톡에서 공유한 자료가 묻히죠. SQUIZ는 이 모든 흐름이 하나로 연결됩니다. 모임에서 나온 핵심 내용이 자동 정리되고, 그걸 기반으로 퀴즈가 출제되는 식이에요."

### Q2. "무료인가요?"

> "현재 모든 기능이 무료로 제공됩니다."

### Q3. "앱이 iOS에서도 되나요?"

> "현재는 Android 앱을 지원하고 있고, iOS는 향후 로드맵에 포함되어 있습니다. 웹 버전은 모든 기기에서 접속 가능합니다."

### Q4. "블루투스 출석이 조작될 수 있지 않나요?"

> "블루투스 신호는 약 10m 범위로 제한되어 있어서, 실제로 근처에 있어야만 인식됩니다. GPS보다 오히려 실내에서 더 정확하고, 건물 밖에서 출석하는 것은 물리적으로 어렵습니다."

### Q5. "AI가 만든 학습 계획이 정확한가요?"

> "AI가 만든 계획은 초안입니다. 바로 쓸 수도 있고, 자유롭게 수정할 수도 있습니다. 핵심은 '빈 종이에서 시작하는 부담'을 없애주는 것입니다. 초안이 있으면 수정하는 건 훨씬 쉽죠."

### Q6. "퀴즈 복습이 실제로 효과가 있나요?"

> "저희가 적용한 건 '간격 반복'이라는 학습법입니다. 에빙하우스의 망각 곡선 연구에서 시작된 방법이고, Duolingo 같은 유명 학습 앱에서도 같은 원리를 사용합니다. 잊어버리기 직전에 다시 보면 기억이 오래 유지된다는 원리예요."

### Q7. "몇 명이나 동시에 화상 모임할 수 있나요?"

> "한 모임에 최대 6~8명까지 쾌적하게 참여할 수 있습니다. 일반적인 스터디 그룹 규모에 최적화했습니다."

## 9. 발표 전략 (일반인 대상 특화)

### 핵심 원칙

1. **기술 용어 금지** — "WebRTC SFU"가 아니라 "여러 명이 동시에 화상으로 만남". "FSRS 알고리즘"이 아니라 "잊어버리기 직전에 다시 출제"
2. **체감 가능한 비교** — "이전에는 이랬는데, 이제는 이렇게 됩니다" 형식
3. **감정에 호소** — "이런 경험 있으시죠?" → 공감 → 해결
4. **숫자는 최소한** — "70% 망각"처럼 직관적인 것만 사용

### 데모 순서 (일반인 최적화)

| 순서 | 기능 | 이유 |
|------|------|------|
| 1 | **AI 학습 계획** | 실시간 생성되는 게 시각적으로 인상적 |
| 2 | **퀴즈 학습 전체** | 가장 많은 시간 할애. 복습→약점분석→코스→게임→대회 순 |
| 3 | **화상 모임 + AI 정리** | 실용적 가치가 즉시 와닿음 |
| 4 | **워크스페이스 + 캘린더** | 빠르게 보여주기 |
| 5 | **블루투스 출석 (앱)** | 모바일 전용 기능으로 차별화 마무리 |

### 주의사항

- "아키텍처", "프로토콜", "SFU" 같은 단어 사용 금지
- "블루투스 출석" → OK, "BLE iBeacon" → NG
- "AI가 자동으로" → OK, "GPT 스트리밍 + S-ROBERTA 임베딩" → NG
- 데모 중 로딩이 길면 "잠시 기다려주세요" 대신 미리 준비된 결과 화면 보여주기
