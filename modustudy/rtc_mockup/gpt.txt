페이지는 HTTPS인데, SFU(socket.io)는 ws://(비보안)로 열려 있어서
Chrome이 Mixed Content로 차단한 것이다.

에러 메시지가 그대로 말해줘 👇

HTTPS 페이지 → ws://192.168.100.90:4000
❌ 반드시 wss:// 여야 함

지금 구조를 정확히 쓰면
브라우저
https://192.168.100.90:3000   ✅ HTTPS

SFU (socket.io)
ws://192.168.100.90:4000     ❌ HTTP WebSocket


👉 HTTPS 페이지에서는

ws:// ❌

wss:// ✅

이건 CORS 문제가 아니라 브라우저 보안 정책이야.

❗ 중요한 현실 정리 (이게 핵심)
❓ “그럼 ws://는 아예 못 쓰는 거야?”

👉 HTTPS 환경에서는 맞다.

페이지	WebSocket
http://	ws:// 가능
https://	❌ ws:// 불가
https://	✅ wss:// 필수
✅ 해결 방법은 딱 3가지 (우선순위 순)
🟢 해결책 1️⃣ (정석 / 실서비스 방식 ⭐⭐⭐⭐⭐)
👉 SFU(socket.io)도 HTTPS(WSS)로 만든다

즉:

ws://192.168.100.90:4000  ❌
wss://192.168.100.90:4000 ✅

SFU(Node.js)에서 HTTPS 서버로 socket.io 띄우기
import https from "https";
import fs from "fs";
import { Server } from "socket.io";

const server = https.createServer({
  key: fs.readFileSync("192.168.100.90-key.pem"),
  cert: fs.readFileSync("192.168.100.90.pem"),
});

const io = new Server(server, {
  cors: { origin: "*" }
});

server.listen(4000);

React에서는
const socket = io(`wss://${window.location.hostname}:4000`);


✔ LAN OK
✔ AWS OK
✔ 실서비스 OK

👉 이게 정답

🟡 해결책 2️⃣ (개발 편의용, 임시)
👉 React를 다시 HTTP로 내린다 (비추천)
HTTPS=false

👉 그래서 답은 1번뿐

왜 이 문제는 “캠 켤 때만” 터졌냐면

아주 정확한 이유가 있어 👇

단계	통신
방 입장	Spring Boot + SockJS (proxy, HTTPS)
캠 ON	SFU socket.io (4000)

👉 SFU만 아직 ws:// 였던 것

최종 체크리스트 (이거 되면 끝)

 SFU 서버를 https.createServer로 띄움

 mkcert 인증서를 SFU에서도 사용

 socket.io 주소를 wss://로 변경

 localhost 완전 제거

 다른 PC에서도 동일 동작 확인

교수님 / 면접에서 이 상황 설명하는 한 문장 🎯

“HTTPS 환경에서 Mixed Content 정책으로 인해
SFU의 WebSocket(ws) 연결이 차단되었고,
SFU 서버를 HTTPS 기반(WSS)으로 전환하여 해결했습니다.”
